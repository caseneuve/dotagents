## Summary

Slice 2 rewrites the 4 pk-tmux shell scripts (331 lines) into babashka with the same clean FCIS split established in slice 1. The pure core is well-tested (185-line test file, 9 test groups). The CLI layer is the most complex in the project so far — `cmd-run` alone is ~80 lines of I/O orchestration. A custom test container image adds tmux for E2E. ## Findings

### Critical

- [ ] **cli.clj:125–126** — **Shell injection via `--cd`.** The `cd` argument is sent unsanitized into tmux via `send-keys`:
  ```clojure
  (tmux! sock "send-keys" "-t" target (str "cd " cd) "Enter")
  ```
  A `--cd` value containing shell metacharacters (`;`, `$()`, backticks) would execute arbitrary commands in the tmux pane. This is the same pattern the shell version had, but in agentic use the `--cd` value may come from LLM output. Wrap in single-quotes: `(str "cd '" cd "'")`, or validate that `cd` is a simple path.

### Important

- [ ] **cli.clj:135** — **Command injection via `command` arg.** Similarly, the command string is interpolated into a `send-keys` call:
  ```clojure
  (str "echo " start "; " command "; echo " end ":$?")
  ```
  This is intentional (the whole point is running the command), but the marker echoes assume the command doesn't contain the marker strings. If a command's output happens to contain a line matching `TMUXRUN_..._END:N`, the extraction will pick the wrong boundary. Low probability but worth a defensive note in the docstring.

- [ ] **cli.clj:67** — **clj-kondo warning: single-arg `and`.**
  ```clojure
  (if (and (tmux? sock "has-session" "-t" session))
  ```
  The `and` wrapping a single expression is a no-op. Should be just `(if (tmux? ...)`.

- [ ] **core.clj:213** — **clj-kondo warning: unused binding `project`** in `format-create-output`. The destructuring binds `project` but never uses it. Either remove it from the destructuring or use it (e.g., in the output string).

- [ ] **bb.edn** — **No `build:test-image` task.** The `test:unit` and `test:e2e` tasks now use `localhost/dotagents-test` (which requires tmux), but there's no bb task or documented step to build this image from `test/Containerfile`. A first-time contributor running `bb test` will get an image-not-found error with no guidance. Add a `build:test-image` task or a pre-check.

- [ ] **bb.edn:24** — **unit-runner classpath string keeps growing.** Now 120+ chars with 6 path segments. This was flagged in the previous review as a slice-6 concern — still tracking. The E2E runner doesn't have this problem because it doesn't directly load skill code.

### Minor

- [ ] **cli.clj:290–292** — **Dispatch table formatting.** The closing `}` for the first map entry is on the same line as the next entry's `{:cmds`, making the table hard to scan:
  ```clojure
  [{:cmds ["create"] :fn cmd-create
   :spec {...} :args->opts [...]}
   {:cmds ["run"] :fn cmd-run}
  ```
  Each entry should start on its own line.

- [ ] **cli.clj:143** — **`(Thread/sleep 2000)` polling interval is hardcoded.** The shell version also used 2s. For short commands in E2E, this means at least a 2s floor before detecting completion. The timeout parameter is configurable but the poll interval isn't. Not a bug, but a future improvement opportunity.

- [ ] **core.clj:108** — **`parse-run-args` silently drops extra positional args.** If someone passes `run win cmd extraarg`, `extraarg` replaces `:command`. This matches shell behavior (last positional wins), but it's arguably surprising. Consider joining extra positionals into the command string, since multi-word commands are common: `run build "make -j4"` works but `run build make -j4` (unquoted) would lose `make`.

- [ ] **core.clj:100–111** — **`parse-run-args` doesn't handle `--timeout` without a value.** If someone passes `run win cmd --timeout` (missing value), `(first tail)` is nil and `(parse-long nil)` returns nil, producing `{:timeout nil}`. The poll loop compares `(>= elapsed timeout)` which would NPE. Add a nil check or default.

- [ ] **test/Containerfile** — Minimal and correct, but consider pinning the base image tag for reproducibility: `FROM docker.io/babashka/babashka:1.x.y`.

- [ ] **test/e2e/cases.edn** — The E2E test uses `bash -c` with a long inline command string. Per AGENTS.md: "prefer `test/fixtures/` over giant inline shell blobs." The current strings are borderline but manageable. Worth watching as more E2E tests are added.

- [ ] **core_test.clj (add-todo):229** — clj-kondo info: `(str "body")` — single arg to `str` is already a string. Trivial.

## Test Coverage Assessment

**Unit tests (core.clj):** Excellent. All pure functions are covered:
- MD5 hashing with known values (✓ verified against `echo -n hello | md5sum`)
- Session info derivation: determinism, different branches, socket paths
- Prefix detection: env, claude path heuristic, default
- Run arg parsing: positional, named, defaults, missing command
- Marker generation
- Output extraction: 7 cases including scrollback, empty output, missing markers, multi-digit exit codes
- Status formatting: no-session, active with windows
- Window busy detection

**Gaps:**
- No unit test for `parse-run-args` with missing `--timeout` value (NPE risk)
- No unit test for `parse-run-args` with extra positional args
- No test for `format-create-output` `:exists` path (only `:created` is exercised indirectly via E2E)
- E2E covers create + run + idempotent create, but not status or wait subcommands
- No E2E test for timeout behavior (would need a slow command + short timeout)

## Security Assessment

**Shell injection via `--cd`** is the primary concern (see Critical finding). The command argument itself is necessarily passed to the shell, but the `--cd` path should be sanitized or quoted since it's expected to be a simple directory path.

## Positive Observations

- FCIS split remains clean: `core.clj` has zero imports of `babashka.process` or `java.io`
- Output extraction logic is robust — the "last occurrence" strategy correctly handles tmux scrollback
- `derive-session-info` vs `derive-simple-session-info` is a clean separation of branch-aware (run) vs simple (create/status) session naming
- Custom test image is the right approach — keeps tmux dependency contained
- E2E test includes cleanup (`kill-server`) even on failure via `; rc=$?; ...; exit $rc` pattern
- Test-to-code ratio stays strong: 185 test lines / 221 core lines
