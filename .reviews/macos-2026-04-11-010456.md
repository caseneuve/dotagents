### Important

- [ ] **shared/skills/add-todo/src/todo/cli.clj:88–90** — `new-spec`'s `:error-fn` creates a new `Exception` just to check `ex-data`, which will always be nil. The `error-fn` callback receives a plain map from `babashka.cli` with keys `:cause`, `:msg`, `:option` — it doesn't receive an exception. The guard `(= :org.babashka/cli (:type (ex-data (Exception.))))` will never match, so all CLI parse errors silently fall through without printing anything. Fix: remove the ex-data check and act on the map directly:
  ```clojure
  :error-fn (fn [{:keys [cause msg option]}]
              (binding [*out* *err*]
                (println (str "Error (" (name cause) "): " option " — " msg)))
              (System/exit 1))
  ```

- [ ] **shared/skills/add-todo/src/todo/core.clj:61–64** — `extract-id` uses `#"^[\d.]+"` which will match a leading `.` (e.g., filename `.5-bad.md` → `.5`). This is unlikely in practice but the regex should be tightened to `#"^\d[\d.]*"` to require a leading digit. The same regex also matches trailing dots (`0001.`) — consider `#"^\d+(?:\.\d+)*"`.

- [ ] **bb.edn:2–3, 17** — The root `bb.edn` now hard-codes paths into `shared/skills/add-todo/{src,test}`. As more slices land (tmux-agent, review, detect-lint, sandbox), this will grow unwieldy. Consider a convention like adding each skill's paths via a shared prefix glob or a dynamic loader, before more slices land.

- [ ] **shared/skills/add-todo/src/todo/core.clj:56** — `parse-frontmatter` regex `#"^([a-z][-a-z]*): *(.*)"` only matches lowercase-letter keys with hyphens. The `blocked-by` and `blocks` keys match, but if any future field uses digits or underscores (e.g., `blocked_by`), it won't parse. The existing shell scripts use a broader pattern. Low risk but worth noting.

### Minor

- [ ] **shared/skills/add-todo/src/todo/cli.clj:67–68** — `cmd-list` exits 0 when the directory doesn't exist, which is consistent with the shell version but makes it impossible for callers to distinguish "no directory" from "directory exists, 0 results". Consider exit code 0 but printing a warning on stderr (already done) — this is fine as-is for now but worth a design note.

- [ ] **shared/skills/add-todo/src/todo/core.clj:167–168** — `next-id` with an empty `filenames` and no parent returns `"0001"`, which is correct. But with an empty `filenames` and a parent, it returns `"PARENT.1"` — this could create an orphan sub-task if the parent doesn't exist. The shell version has the same behavior. Consider validating parent existence in the CLI layer.

- [ ] **shared/skills/add-todo/todo-status.sh:62–68** — The macOS compat fix shells out to `sed --version` to detect GNU vs BSD sed, but `sed --version` writes to stderr on GNU and exits non-zero on BSD. The `&>/dev/null 2>&1` redirect is redundant (both fd1 and fd2 redirected twice). Simplify to `if sed --version >/dev/null 2>&1; then`.

- [ ] **shared/skills/pk-tmux/tmux-run.sh:77** — The fallback `md5 -qs` is macOS-specific. The `||` chain `md5sum 2>/dev/null | head -c 6 || md5 -qs "$BRANCH" | head -c 6` is fragile because the `|` to `head` makes the exit code come from `head`, not `md5sum`. If `md5sum` is missing, the pipe still succeeds (empty output through `head`), so the `||` fallback never fires. Wrap in a subshell or use an if/else like the `sed` fix.

- [ ] **docs/bb-rewrite-plan.md:1** — The plan references `Branch: macos` but doesn't mention the macOS compat shell fixes as a prerequisite. The shell scripts still exist alongside the new bb code. Worth a note in the plan about the deprecation/removal timeline for the `.sh` files.

- [ ] **shared/skills/add-todo/src/todo/core.clj:144** — `normalize-labels` throws on invalid chars but doesn't document which CLI layer catches this. In `cmd-new`, there's no try/catch around the call, so an invalid label will produce an unhandled exception with a stack trace instead of a clean error message.

- [ ] **test/e2e/cases.edn** — The `0002-no-labels.md` fixture has no `labels` key in frontmatter. The Babashka `parse-frontmatter` returns `nil` for missing keys, while the shell version would return empty. The `list` E2E test filters by label on a different fixture, so this inconsistency isn't tested. Consider adding a test that lists with `--label` where some todos have no `labels` key at all.
