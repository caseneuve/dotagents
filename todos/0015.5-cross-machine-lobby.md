## Context

Currently lobby names are derived from local env (cmux workspace, tmux session).
Different machines produce different hashes even for "the same" workspace. Need
a stable lobby name that works across machines pointing at the same relay.

## Acceptance Criteria
- [ ] `resolveLobby()` supports relay-scoped lobbies: `relay/${hash_of_server_url}`
- [ ] `AGENT_LOBBY` env var overrides all auto-detection
- [ ] Same-machine lobbies unchanged (cmux/tmux derivation still works)
- [ ] Agents on different machines pointing at same relay auto-discover each other
- [ ] Documented in README

## Affected Files
- `pi/extensions/agent-channel/index.ts` — update `resolveLobby()`
- `pi/extensions/agent-channel/README.md` — cross-machine setup docs

## Notes
- Priority chain: AGENT_LOBBY env > relay-scoped > cmux workspace > tmux session > file/lobby
- Relay-scoped lobby only applies when HttpTransport or UdsTransport is active
- Consider: should the relay server advertise a lobby name? (probably YAGNI)
