## Context

Manual multi-agent setup is possible but tedious. This task adds a convenience
path where one agent (or user command) can spawn another interactive Pi process
in tmux, preconfigured to join a communication channel and follow a role
prompt.

## Acceptance Criteria

- [ ] There is a command or tool that spawns a new interactive Pi process in a
      tmux pane/window with channel + role boot instructions.
- [ ] Spawned agent startup is deterministic enough for repeated use (session,
      cwd, channel metadata, prompt directive).
- [ ] The main frame reports where the spawned process lives (session/window) so
      the human can inspect it directly.
- [ ] Spawn failures are surfaced clearly and do not leave ambiguous state.
- [ ] E2E coverage verifies spawned agent can send/receive over the same
      communication channel.

## Affected Files

- `pi/extensions/agent-comm/` — spawn command/tool integration
- `shared/` or `scripts/` — helper script if tmux orchestration is shared
- `pi/README.md` — spawn flow documentation
- `test/e2e/` — spawn + communication checks

## E2E Spec

GIVEN an orchestrator agent in Pi
WHEN it spawns a reviewer/worker agent in tmux communication mode
THEN the spawned agent appears in a known tmux target and can exchange channel
messages with the orchestrator

## Notes

Treat tmux as an inspectability and convenience layer, not the message backend.
