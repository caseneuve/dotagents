## Context

Central relay that agents connect to for push-based message delivery. Runs on
Raspberry Pi (or any machine). Listens on both UDS (local) and HTTP (remote).
Channels are the only addressing primitive — server is a dumb keyed pub/sub broker.

## Acceptance Criteria
- [ ] Server listens on UDS socket (configurable path, default `/tmp/agent-channels.sock`)
- [ ] Server listens on HTTP port (configurable, default 7700)
- [ ] `POST /channels/:channel/messages` — publish, fans out to all subscribers
- [ ] `PATCH /channels/:channel/messages/:id` — ack
- [ ] `GET /channels/:channel/stream` — SSE subscription, pushes new messages live
- [ ] `GET /channels` — list all channels with message counts
- [ ] UDS protocol: JSON frames with `{action, channel, msg?}` structure
- [ ] Messages stored in memory with configurable max per channel
- [ ] Server is a single Bun/TypeScript entrypoint

## Affected Files
- `relay/server.ts` — new, relay server entrypoint
- `relay/store.ts` — new, in-memory channel store
- `relay/server.test.ts` — new, tests

## Notes
- Bun has native UDS support via `Bun.listen` with `unix` option
- Bun has native HTTP server — no framework needed
- SSE is just `Transfer-Encoding: chunked` with `text/event-stream` content type
- In-memory store is sufficient — agents reconnect and catch up via `Last-Event-ID`
- Consider optional disk persistence later (YAGNI for now)
