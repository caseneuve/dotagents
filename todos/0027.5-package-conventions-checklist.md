## Context

The seven move-only extractions need a small shared checklist, not a generalized package framework
or an eighth template repository. The conventions should live in a canonical document in dotagents
and state only what is needed to move, install, test, review, and maintain the packages consistently.

Repositories are public under `github.com/caseneuve/`. They consume the supported
`@earendil-works/*` Pi APIs; they are not Earendil-owned packages. npm names/scopes and publication
are deferred to `0051`.

## Acceptance Criteria

- [ ] Add one canonical package-conventions/checklist document in dotagents and link it from the epic.
- [ ] List the seven canonical GitHub repository URLs, public visibility, local checkout convention under `~/git/pi/`, and new-repository default branch convention.
- [ ] Define repository/manifest package names for local and Git use without reserving an npm scope.
- [ ] Decide and document license/copyright defaults for the new public repositories before code is moved.
- [ ] Require an explicit `pi` manifest with stable resource paths and document the exact Pi filtering/settings mechanism used to enable or disable bundled resources.
- [ ] Verify and document the exact current Pi syntax for an immutable supported Git ref; do not assume tag and SHA behavior without checking current Earendil Pi docs.
- [ ] Define dependency placement needed by Pi installs: Pi APIs as peer dependencies and non-Pi runtime dependencies in the appropriate installable dependency section.
- [ ] Define the minimum README, existing-test migration, formatting, local-install, Git-install, commit/todo linkage, and peer-review checklist for each repository.
- [ ] Define experimental status as explicit manifest exclusion for scaffold work; Tree-sitter remains outside this package set.
- [ ] Keep npm publishing, generalized scaffolding automation, new APIs, and extension improvements out of scope.

## Affected Files

- A canonical packaging conventions document under dotagents docs or `pi/`.
- New repositories under `github.com/caseneuve/` — checklist consumers.

## Notes

Compliance is checked by `0027.9` against the documented checklist. Do not create a dedicated
`pi-package-template` repository as part of this epic.
