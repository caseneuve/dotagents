## Context

The seven move-only extractions need a small shared checklist, not a generalized package framework
or an eighth template repository. The conventions should live in a canonical document in dotagents
and state only what is needed to move, install, test, review, and maintain the packages consistently.

Repositories are public under `github.com/caseneuve/`. They consume the supported
`@earendil-works/*` Pi APIs; they are not Earendil-owned packages. npm names/scopes and publication
are deferred to `0051`.

## ADR decisions

- Canonical guidance lives in `docs/pi-package-conventions.md`; package repositories copy the lean `docs/pi-package-AGENTS.md` as their local `AGENTS.md`.
- New repositories use public GitHub visibility, `master` default branches, MIT licensing, unscoped repository/package names, and `~/git/pi/<repo>` checkouts.
- Pi manifests are explicit and filterable through `settings.json` package objects and `pi config`; experimental resources require manifest-level `!pattern` exclusions. Tree-sitter is excluded.
- Git refs are immutable tags or commit SHAs using Pi's documented `git:host/path@ref` syntax. Runtime dependencies go in `dependencies`; Pi APIs go in `peerDependencies`.
- Each package uses independent CalVer `YYYY.MM.PATCH`, a root `CHANGELOG.md`, and `vYYYY.MM.PATCH` release tags.
- Development follows discussion → todo → task branch → checkpoint commits → tests/hooks → peer review → squash/merge. Prefer TDD (red/green/refactor); use documented manual testing when automation is impractical. Design and review follow FCIS, typed modularity, KISS, DRY, and YAGNI; UI changes require screenshot review unless explicitly waived by a human.
- TypeScript formatting/linting is enforced by committed `prek` hooks. Package READMEs link to https://pi.dev/ and repositories use relevant discovery topics (`pi`, `agents`, `extensions`, plus package-specific terms).

Compliance is checked by `0027.9` against the documented checklist. Do not create a dedicated
`pi-package-template` repository as part of this epic.
