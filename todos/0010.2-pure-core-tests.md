## Context

Once the pure core is extracted into `core.ts`, it should have full unit test coverage.
Pure functions with zero mocks — fast, reliable tests.

Backend wiring (read/ack/publish against real filesystem) also needs coverage to
catch misconnections during the module split. These use a tmpdir — no mocks.

## Acceptance Criteria

### Pure core (`core.test.ts`)
- [x] `ackMessages` tests: all 3 modes (*, last, specific id), returns new object, original unchanged
- [x] `shouldTriggerTurn` tests: presence type, OUT suffix, OVER suffix, no suffix
- [x] `channelPath` tests: safe character encoding, path construction
- [x] `filterMessages` tests: since, unacked, type filtering, combinations

### Backend wiring (`backends.test.ts`)
- [ ] `publish` → file written with correct message (deferred to post-split)
- [ ] `read` → filters delegated correctly (deferred to post-split)
- [ ] `ack` → file updated, returns correct count (deferred to post-split)
- [ ] Tests run against tmpdir, no mocks (deferred to post-split)
- **Note:** Deferred until 0010.4 — backends currently close over module-level
  `CHANNEL_DIR`/`readChannelFile`/`writeChannelFile` with no injection point.
  Testing against `~/.agent-channels` would touch real data. The module split
  makes `CHANNEL_DIR` injectable, at which point these tests become trivial.
