## Context

Once protocol/store exists, agents need a usable interface. This task adds the
extension tools that LLMs call plus human-facing commands for observability and
control in the main Pi frame.

## Acceptance Criteria

- [ ] Extension tools exist for at least send, poll, and ack operations over the
      SQLite-backed communication channels.
- [ ] Tool results are concise and structured so agents can reason over them
      without noisy transcript dumps.
- [ ] Human inspection commands exist to list channels, inspect recent messages,
      and filter by participant or thread.
- [ ] Basic safeguards exist (message size limits, participant validation,
      readable errors).
- [ ] E2E coverage verifies two independently started agents can exchange at
      least one task/result round-trip.

## Affected Files

- `pi/extensions/agent-comm/` — tool and command registration
- `pi/README.md` — usage documentation for communication tools/commands
- `test/e2e/pi/` — communication round-trip checks

## E2E Spec

GIVEN two independently running Pi agents configured with the communication
extension
WHEN agent A sends a message to agent B over a channel
THEN agent B can poll, process, reply, and both messages can be inspected from
commands in the main frame

Keep parent context clean by default; use commands for detailed inspection.
