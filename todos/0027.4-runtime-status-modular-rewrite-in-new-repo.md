## Context

Status-related extensions (`runtime-footer`, `branch-status`, `session-notes`, `bookmark`
interactions) should be extracted and redesigned in a new repo, not refactored in dotagents.

Target architecture is pluggable/composable:
- user-configurable left/right layout
- block registry (core + third-party/custom blocks)
- graceful behavior when optional block providers are missing

## Acceptance Criteria

- [ ] New repo created under `~/git/pi/` for runtime status system.
- [ ] v1 design doc defines plugin/block API and user configuration format.
- [ ] Minimal implementation renders configurable left/right block layout.
- [ ] Core blocks include at least cwd, git/model context, and extension-provided status interop.
- [ ] Package uses `@earendil-works/*` deps and latest Pi API conventions.

## Affected Files

- `pi/extensions/runtime-footer.ts`
- `pi/extensions/branch-status.ts`
- `pi/extensions/session-notes.ts`
- `pi/extensions/bookmark.ts`

## Notes

- Prefer rewrite in new repo if faster/cleaner than direct port.
