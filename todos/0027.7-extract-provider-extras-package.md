## Context

The Bedrock empty-response mitigation and subscription usage overlay are provider/account-specific.
They belong in one selectively enabled package rather than in the global dotagents extension path
or in separate tiny repositories.

## Acceptance Criteria

- [ ] Create `~/git/pi/pi-provider-extras` using the shared package convention.
- [ ] Package `bedrock-empty-response-mitigation` and `usage` as separate filterable extension resources.
- [ ] Missing credentials or an unrelated active provider does not break Pi startup.
- [ ] Each extension can be enabled or disabled independently with Pi package configuration.
- [ ] README documents provider assumptions, credential sources, and local/Git installation.

## Affected Files

- `pi/extensions/bedrock-empty-response-mitigation.ts` — source to migrate.
- `pi/extensions/usage.ts` — source to migrate.
- New `~/git/pi/pi-provider-extras` repository — canonical implementation.

## Notes

Keep this package out of universal defaults unless later usage proves it should be global.
