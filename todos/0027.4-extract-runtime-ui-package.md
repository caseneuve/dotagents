## Context

The runtime UI package should own the status surfaces that must not race: runtime footer,
editor-border status, conversational branch status, and shared Git status formatting. It should be
a global-default package with pluggable blocks and graceful optional integration with separately
installed conversation and agent-channel packages.

Target architecture:

- one footer owner
- one `setEditorComponent` owner
- user-configurable left/right layout
- block registry for core and optional providers
- graceful behavior when optional block providers are missing

## Acceptance Criteria

- [ ] Create `~/git/pi/pi-runtime-ui` using the shared package convention.
- [ ] Preserve current runtime footer, editor status, and branch status behavior before or during the modular redesign.
- [ ] The package has exactly one footer owner and one editor-component owner.
- [ ] A documented block/provider API supports core and optional third-party/custom blocks.
- [ ] Core blocks cover cwd/project, Git, provider/model/thinking/cost/context, and generic extension status interop.
- [ ] Session-notes and agent-channel/comms information appears when available without hard package imports.
- [ ] Current JSONC configuration, aliases, separators, truncation, and formatting modes have a migration/compatibility path.
- [ ] Package uses `@earendil-works/*` dependencies and current Pi conventions.
- [ ] Local-path and pinned Git installs are documented and validated as global defaults.

## Affected Files

- `pi/extensions/runtime-footer.ts` — source and behavior baseline.
- `pi/extensions/editor-status.ts` — sole editor-component owner baseline.
- `pi/extensions/branch-status.ts` — conversational status baseline.
- `pi/extensions/shared/runtime-status-git.ts` — shared Git formatting.
- New `~/git/pi/pi-runtime-ui` repository — canonical implementation.

## Notes

Do not move bookmark or session-notes implementation into this package; they belong to
`pi-conversation-tools`. Runtime UI consumes their optional published status/events instead.
Preserve stable dotagents behavior until package validation and cutover are complete.
