## Context

The default user experience should make subagent workflows understandable
without overwhelming the main conversation. While a workflow runs, the human
should see which round is active, which subagent is currently working, and the
latest meaningful action. After completion, the collapsed tool result should
still provide a useful summary at a glance.

This task is about the compact UX: status line, optional lightweight widget,
and collapsed/expanded inline rendering for the parent-facing workflow tool.

## Acceptance Criteria

- [ ] While a subagent workflow is running, Pi shows compact live progress such
      as active round, active agent, and latest summarized action.
- [ ] The collapsed renderer for a completed workflow shows a concise outcome
      such as pass/fail, rounds used, findings fixed or remaining, and high-
      level usage stats.
- [ ] The expanded renderer shows structured round summaries derived from the
      trace model rather than dumping raw nested transcript text.
- [ ] Live updates are meaningful and throttled/coalesced enough to avoid noisy
      flicker from every token or verbose tool output.
- [ ] The UX behaves sensibly for success, failure, and abort cases.
- [ ] The renderer and status behavior are documented with enough detail to use
      as the canonical pattern for future subagent workflows.

## Affected Files

- `pi/extensions/subagents/index.ts` — tool registration and wiring
- `pi/extensions/subagents/render.ts` — collapsed/expanded tool rendering
- `pi/extensions/subagents/status.ts` — footer/widget progress updates
- `pi/extensions/subagents/trace.ts` — helpers that derive compact display data
- `pi/README.md` — document the workflow UI behavior if user-facing behavior
  changes materially
- `test/unit/pi/subagents/` — formatter and summary derivation tests
- `test/e2e/pi/` — inspectability checks for rendered workflow states

## E2E Spec

GIVEN a running `review_loop` workflow
WHEN the user watches the tool execution in Pi
THEN they see compact live updates with current round, active subagent, and the
latest summarized action

GIVEN a completed workflow
WHEN the user views the collapsed and expanded tool result
THEN the collapsed view stays concise and the expanded view shows structured
round summaries suitable for human inspection

## Notes

Keep the main-agent context clean: this task is about human-visible rendering,
not about returning giant transcripts to the parent model. Prefer summary rows
and round sections over chat-bubble-style duplication.
