## Context

Agent identity is spread across `agentId`, `agentLabel`, `agentName()`, and
`CMUX_AGENT_NAME` env var with a priority chain. Works but hard to reason about.

## Acceptance Criteria
- [x] Single `AgentIdentity` structure: `{ id: string, label?: string, env?: string }`
- [x] `resolve()` method with clear priority: env > label > id
- [x] Persist/restore as one unit via `appendEntry`
- [x] `agentName()` replaced with `identity.resolve()`

## Affected Files
- `pi/extensions/agent-channel/identity.ts` — new file
- `pi/extensions/agent-channel/index.ts` — use new structure

## Notes
- Priority chain: CMUX_AGENT_NAME env > user-set label > session-generated id
- Must remain compatible with `/agent-name` command and system prompt injection
