## Context

Before adding tests, decide which helpers should stay as shell entrypoints with end2edn coverage and which should be rewritten or partially extracted into Babashka for direct unit tests under `bb test`.

A first pass from the script review:

- Likely keep as I/O-focused scripts and cover with end2edn: `todo-*.sh`, `review-file.sh`, `sandbox-*.sh`, `tmux-*.sh`, `smart-lint.sh`, `statusline-command.sh`
- Likely good direct `bb test` target already: `shared/skills/org-journal/new-entry.bb`
- Possible hybrid approach: extract pure decision logic from larger shell scripts into Babashka, keep thin shell wrappers at the boundary

## Acceptance Criteria

- [ ] Each helper script is listed in a strategy matrix with its proposed test approach.
- [ ] The matrix identifies where pure logic extraction to Babashka would materially simplify testing.
- [ ] The plan names the concrete test files or suites to add under `test/unit/` and `test/e2e/`.

## Affected Files

- `shared/skills/add-todo/*.sh` - decide shell contract vs extracted logic coverage
