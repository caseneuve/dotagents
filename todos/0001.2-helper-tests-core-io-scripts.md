## Context

Several helpers are mostly filesystem and git I/O boundaries, so they are good candidates for end2edn-style contract tests in isolated temp repos rather than deep rewrites:

- `shared/skills/add-todo/todo-next-id.sh`
- `shared/skills/add-todo/todo-new.sh`
- `shared/skills/add-todo/todo-list.sh`
- `shared/skills/add-todo/todo-status.sh`
- `shared/skills/code-review/review-file.sh`
- `shared/skills/sandbox/sandbox-create.sh`
- `shared/skills/sandbox/sandbox-finish.sh`

## Acceptance Criteria

- [ ] end2edn coverage verifies the add-todo scripts against realistic todo directory fixtures and parseable stdout contracts.
- [ ] end2edn coverage verifies review-file behavior for branch naming, latest/list lookup, and error cases.
- [ ] end2edn coverage verifies sandbox create/finish behavior in disposable git repos or clearly documents the minimal cases that remain untested.
- [ ] Tests run through the canonical `bb test` / `bb test:e2e` entrypoints.

## Affected Files

- `shared/skills/add-todo/*.sh` - todo CLI contracts
- `shared/skills/code-review/review-file.sh` - review file path and branch prefix logic
- `shared/skills/sandbox/*.sh` - worktree lifecycle behavior
- `test/e2e/cases.edn` - add high-level workflow cases
- `test/e2e/assertions.clj` - add assertions only if reusable
- `test/fixtures/...` - readable git/todo fixtures when shell setup becomes noisy

## Notes

Prefer assertions on resulting repo/todo state over brittle output scraping. Use `/tmp` inside the container for mutable test state and avoid touching the host environment.
