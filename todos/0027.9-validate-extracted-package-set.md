## Context

Verify that the move-only packages reproduce the current dotagents-loaded resource set before
bootstrap adopts them. Validation checks packaging and parity, not extension improvements.

## Acceptance Criteria

- [ ] Record the pre-cutover inventory of extension entrypoints, tools, commands, shortcuts, settings paths, skills, prompts, themes, and relevant current platform limitations.
- [ ] For every package, run the moved existing tests and the canonical package checklist command(s) documented by `0027.5`.
- [ ] Test local-path installation and Git installation using the exact documented immutable Pi-supported ref syntax in an isolated Pi config/dependency state.
- [ ] Confirm explicit manifest resources appear in `pi list`/`pi config` and can be filtered using the documented mechanism without changing their stable resource identity.
- [ ] Compare the installed package inventory and representative behavior to the pre-cutover baseline.
- [ ] Compose intended global/selective packages without duplicate commands, tools, footer owners, or editor owners.
- [ ] Verify `/reload`, startup, and shutdown have no new diagnostics caused by the move.
- [ ] Record repository URLs, tested refs, commands, results, and peer-review evidence for each repository.
- [ ] File non-mechanical improvements separately; they do not block parity validation unless the move caused the regression.

## Affected Files

- New repositories under `~/git/pi/`.
- Package tests/READMEs and the `0027.5` checklist.
- `pi/*` as the behavior baseline.
