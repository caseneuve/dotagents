## Context

Before building workflow UX, we need a canonical data model for subagents and
for the traces they emit. The current example only covers a small frontmatter
shape (`name`, `description`, `tools`, `model`) and subprocess execution. For a
framework, we need a richer Markdown contract that can describe scope,
permissions, and structured handoff expectations, plus a normalized runtime
trace shape that downstream renderers and commands can consume.

This task should establish the contract that later work builds on, not just a
one-off implementation detail hidden inside one extension file.

## Acceptance Criteria

- [ ] A Markdown/frontmatter schema is defined for subagents, covering at least
      name, description, model, allowed tools, and scope/policy fields needed
      for SDK-backed execution.
- [ ] The framework has one normalized in-memory trace shape for workflow runs,
      rounds, handoffs, transcript events, parsed results, and usage stats.
- [ ] Scope rules explicitly define what a subagent may inherit or suppress,
      such as AGENTS files, skills, prompts, extensions, and working directory
      behavior.
- [ ] Tool policy rules explicitly define how read-only vs write-capable agents
      are represented, including a path for constrained bash behavior.
- [ ] Pure parsing/normalization helpers are separated from I/O and are covered
      directly by unit tests.
- [ ] The design is documented clearly enough that later tasks can implement
      renderers and orchestration without changing the schema ad hoc.

## Affected Files

- `pi/extensions/subagents/schema.ts` — frontmatter and trace model types
- `pi/extensions/subagents/registry.ts` — Markdown discovery and normalization
- `pi/extensions/subagents/scoped-loader.ts` — scope/filter contract for
  resources
- `pi/extensions/subagents/tool-policy.ts` — tool-policy interpretation rules
- `pi/README.md` — canonical design notes and schema documentation, with any
  extension-local README used only for narrow implementation notes if needed
- `test/unit/pi/subagents/` — schema parsing, normalization, and trace-model
  tests

## E2E Spec

GIVEN a Markdown subagent definition with scope and policy frontmatter
WHEN the framework loads the definition
THEN it produces a normalized agent config that downstream session and
rendering code can use consistently

GIVEN a workflow round with tool calls, parsed review output, and usage stats
WHEN the framework records the result
THEN it persists a normalized trace object that later UX layers can render
without re-parsing raw assistant text

## Notes

Favor a small but extensible schema. Do not overfit the first version to only
worker/reviewer, but do make the review-loop requirements explicit enough to
avoid fragile freeform parsing later.
