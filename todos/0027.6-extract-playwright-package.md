## Context

The Playwright extension is a mature, heavily used but dependency-heavy and security-sensitive
capability. It should be independently installable and project-local by default rather than loaded
in every Pi session through the dotagents extension directory.

## Acceptance Criteria

- [ ] Create `~/git/pi/pi-playwright` using the shared package convention.
- [ ] Move the complete extension, policy UI/storage, tests, runtime dependency, and documentation.
- [ ] Local-path and pinned Git installs register the full Playwright tool set.
- [ ] Fresh installation handles runtime dependencies reproducibly and documents browser installation.
- [ ] The restrictive allowlist policy and project-local policy storage remain intact.
- [ ] Session shutdown reliably closes browser resources.
- [ ] README recommends project-local installation and documents Linux/macOS expectations.

## Affected Files

- `pi/extensions/playwright/*` — source to migrate.
- New `~/git/pi/pi-playwright` repository — canonical implementation.

## E2E Spec

GIVEN a trusted project with the package installed project-locally
WHEN Pi opens an allowed local URL and uses DOM, screenshot, and console tools
THEN the browser workflow succeeds and disallowed navigation remains blocked by policy.

## Notes

Do not include this package in universal bootstrap defaults. Preserve current tool names so agent
instructions and existing workflows remain compatible.
