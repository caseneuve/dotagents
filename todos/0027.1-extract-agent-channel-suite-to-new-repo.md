## Context

Move the existing agent-channel ecosystem into the public
`github.com/caseneuve/pi-agent-channel` package. Preserve the extension, bundled agent-comms skill,
and current detached relay implementation/management behavior. This task does not redesign or
harden the relay.

## Acceptance Criteria

- [ ] Create the public repo and local checkout at `~/git/pi/pi-agent-channel` using the shared checklist.
- [ ] Move the current extension, tests, bundled `agent-comms` skill, relay implementation, and existing relay management behavior required by the suite.
- [ ] Preserve current file, UDS, and HTTP transport behavior and current cmux/tmux/file display behavior.
- [ ] As a permitted mechanical packaging adaptation, expose the moved relay entrypoint and equivalent current start/stop/status/logs/restart operations from the package; do not add new lifecycle semantics.
- [ ] Preserve existing relay config, socket, log, and process behavior; dotagents `bb relay:*` tasks may remain temporary compatibility wrappers that delegate to the package until cutover.
- [ ] Package resources and the mechanically relocated relay operations are reachable after local-path and Git installation without requiring dotagents implementation files or source symlinks.
- [ ] Existing agent-channel and relay tests pass with documented commands.
- [ ] Package uses the supported `@earendil-works/*` APIs and is suitable as a global default.
- [ ] Any desired PID handling, stale-process recovery, daemon platform integration, health UX, or upgrade improvements are recorded in follow-up `0049`, not implemented here.

## Affected Files

- `pi/extensions/agent-channel/*` — source to move.
- `shared/skills/agent-comms/*` — skill to bundle canonically with the package.
- `shared/relay/*` — implementation to move into the package.
