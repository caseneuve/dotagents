## Context

Move the existing agent-channel ecosystem into the public
`github.com/caseneuve/pi-agent-channel` package. Preserve the extension, bundled agent-comms skill,
and current detached relay implementation/management behavior. This task does not redesign or
harden the relay.

The package must additionally provide a standalone, Bun-backed relay-control CLI named
`pi-agent-channel-relay` with `start`, `stop`, `status`, `restart`, and `logs` operations. An
explicit install/uninstall workflow places or removes its launcher in `~/.local/bin`; Pi package
installation itself must not mutate `PATH`. Bun remains the relay runtime—porting the relay daemon
to Node is out of scope.

## Acceptance Criteria

- [ ] Create the public repo and local checkout at `~/git/pi/pi-agent-channel` using the shared checklist.
- [ ] Move the current extension, tests, bundled `agent-comms` skill, relay implementation, and existing relay management behavior required by the suite.
- [ ] Preserve current file, UDS, and HTTP transport behavior and current cmux/tmux/file display behavior.
- [ ] As a permitted mechanical packaging adaptation, expose the moved relay entrypoint and equivalent current start/stop/status/logs/restart operations from the package; do not add new lifecycle semantics.
- [ ] Provide `pi-agent-channel-relay` as the package-owned, Bun-backed CLI for those operations, with documented help and exit behavior.
- [ ] Provide explicit relay-CLI install and uninstall commands that add/remove its launcher at `~/.local/bin/pi-agent-channel-relay`; Pi package installation does not write to `PATH` or install the launcher.

## Notes

Do not improve relay lifecycle semantics in this task. Follow-up `0049` owns that work.

### 2026-07-11 MVP packaging decision

The initial public release will preserve the Bun-backed relay and explicitly support relay CLI installation only from a retained local checkout at a conventional path. Git-installed extension and skill resources remain required, and file transport remains available without the relay. Portable npm CLI publication, arbitrary installation paths, argument-safe detached spawning, and the Bun-versus-Node runtime evaluation moved to the new repository's todo `0001-publish-relay-cli-to-npm.md`. This is an agreed scope reduction from the original Git-installable relay-operations criterion, not evidence that the original workflow was completed.
