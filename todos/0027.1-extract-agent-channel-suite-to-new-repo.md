## Context

`agent-channel` is tightly coupled with the `agent-comms` skill and relay transport
story. We want one installable suite in a dedicated repo, with detached relay daemon
operations (start/stop/status/health/logs, config, auditing).

## Acceptance Criteria

- [ ] New repo scaffolded under `~/git/pi/pi-agent-channel`.
- [ ] Pi package includes extension + bundled `agent-comms` skill (no symlink dependency on dotagents).
- [ ] Relay component runs detached from any single Pi instance.
- [ ] Relay CLI supports at least: `start --detach`, `stop`, `status`, `health`, `logs`, `restart`.
- [ ] Relay config supports socket path and HTTP host/port.
- [ ] Package uses `@earendil-works/*` deps and current Pi package conventions.

## Affected Files

- `pi/extensions/agent-channel/*` — source to migrate/adapt.
- `shared/skills/agent-comms/SKILL.md` — source to bundle with package.
- `shared/relay/*` and `bb relay:*` behavior — reference for detached relay UX.

## E2E Spec

GIVEN a clean machine with Pi installed
WHEN I install the package and start the relay daemon in detached mode
THEN Pi can use channel tools with relay transport
AND I can manage the relay with start/stop/status/health/logs commands independent of Pi process lifecycle.

## Notes

- Do not tie relay lifecycle to Pi process lifecycle.
- Keep file transport fallback available as zero-config backup.
