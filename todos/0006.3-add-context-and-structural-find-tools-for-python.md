## Context

Once the parser and outline path work, we should test whether Tree-sitter helps
with real agent workflows beyond file summaries. The next two capabilities with
the highest likely payoff are:

- context lookup: given a file position, identify the enclosing syntax path and
  nearest meaningful declaration
- structural find: provide curated query modes such as functions, classes,
  imports, tests, or decorated functions

These tools should stay deliberately narrow and predictable so we can learn
what agents actually use before exposing a more raw query interface.

## Acceptance Criteria

- [ ] A `context`-style tool can return the enclosing syntax chain for a Python
      file position, including the nearest named declaration and its range.
- [ ] A `find`-style tool supports a small curated set of Python query modes
      that are demonstrably useful for agent workflows.
- [ ] Tool outputs are compact, stable, and easy for an agent to consume
      without re-parsing large text blobs.
- [ ] Pure context-path formatting and structural-match shaping helpers are
      covered by unit tests.
- [ ] The implementation reuses the adapter/core boundaries defined in 0006.1
      rather than baking Python-specific assumptions into generic modules.
- [ ] At least one real usage example demonstrates how these tools reduce
      unnecessary full-file reading compared with plain text search alone.

## Affected Files

- `pi/extensions/treesitter/` — context and find tools plus Python query logic
- `pi/README.md` — usage notes if the tool surface becomes user-facing
- `test/unit/pi/treesitter/` — tests for context-path formatting and query-mode
  extraction helpers
- `test/e2e/pi/` — smoke coverage for representative query workflows if added

## E2E Spec

GIVEN a Python file with nested class and function scopes
WHEN the agent asks for syntax context at a specific line and column
THEN the tool returns the enclosing scope chain and nearest declaration with
usable range metadata

GIVEN a Python test file or module with decorated and async functions
WHEN the agent uses the structural-find tool in curated query modes
THEN it receives precise matches for those structures without relying on brittle
regex search

## Notes

Prefer a curated query surface over arbitrary raw Tree-sitter queries for the
first version unless raw-query support turns out to be nearly free and easy to
bound.
