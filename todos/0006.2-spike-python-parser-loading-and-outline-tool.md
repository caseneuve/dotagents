## Context

The first practical question is whether a Pi extension can load a Python
Tree-sitter grammar cleanly and produce useful output with acceptable
complexity. The smallest convincing result is an outline tool that parses a
Python file and returns top-level structure such as imports, classes,
functions, and async functions in a compact agent-friendly shape.

This is the first implementation milestone and the main technical feasibility
spike for the epic.

## Acceptance Criteria

- [ ] The extension can parse at least one Python file successfully inside the
      Pi extension runtime using the chosen Tree-sitter integration approach.
- [ ] A first `outline`-style tool exists and returns compact structured output
      for top-level Python declarations and imports.
- [ ] The output includes stable range information such as start/end lines so
      the agent can use the result to target subsequent reads.
- [ ] Pure outline extraction or output-shaping helpers are covered by unit
      tests rather than relying only on manual smoke testing.
- [ ] Failure modes are handled sensibly, including unsupported file types,
      parse failures, or missing parser dependencies.
- [ ] The spike is small enough to serve as the canonical starting point for
      later Python-specific tools rather than a throwaway experiment.

## Affected Files

- `pi/extensions/treesitter/` — parser initialization, Python adapter, and tool
  registration
- `pi/extensions/treesitter/package.json` — dependencies if the extension is
  packaged as a directory-based extension
- `test/unit/pi/treesitter/` — outline extraction/output-shaping tests for pure
  helpers
- `test/e2e/pi/` — optional smoke coverage for the tool if practical

## E2E Spec

GIVEN a Python module with imports, classes, and functions
WHEN the agent calls the Tree-sitter outline tool on that file
THEN it receives a compact structural summary with declaration kinds, names,
and ranges that can guide follow-up reads and edits

## Notes

Do not try to solve repo-wide search or raw Tree-sitter query UX in this task.
The goal is parser feasibility plus one obviously useful tool.
