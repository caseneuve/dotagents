## Context

Move diff-review, repo-todos, agent-journal, cwd-editor, and attach-screenshot into one public
workbench package. Preserve their current Linux/macOS behavior and limitations; extraction does not
add platform adapters or improve optional-tool handling.

## Acceptance Criteria

- [ ] Create `github.com/caseneuve/pi-workbench` with local checkout `~/git/pi/pi-workbench`.
- [ ] Move all five extensions as separate explicit manifest resources.
- [ ] Preserve `/diff`, all `/diff-*` aliases, revspecs, submodule behavior, overlays, shortcuts, editor export, and screenshot queue behavior.
- [ ] Preserve and accurately document current dependencies and limitations, including Git, `$EDITOR`/`$VISUAL`, `sxiv`, and clipboard commands.
- [ ] Do not claim `attach-screenshot` supports macOS unless the current implementation already does; cross-platform screenshot UX is follow-up `0050`.
- [ ] Move existing tests and make them pass with documented commands.
- [ ] Local-path and immutable-ref Git installs work with supported `@earendil-works/*` APIs.

## Affected Files

- `pi/extensions/diff-review.ts`
- `pi/extensions/repo-todos.ts`
- `pi/extensions/agent-journal.ts`
- `pi/extensions/cwd-editor.ts`
- `pi/extensions/attach-screenshot.ts`
- `pi/extensions/shared/editor-text.ts`
- New `~/git/pi/pi-workbench` repository.

## E2E Spec

GIVEN the same OS and external executables used before extraction
WHEN an existing workbench command or shortcut is used from the installed package
THEN its behavior matches the dotagents-loaded version.
