## Context

Diff review is mature and frequently used through its focused aliases. It belongs in a personal
Linux/macOS workbench package with the repository todo browser, agent journal browser, cwd editor,
and screenshot attachment workflow. These tools share external-editor/repository ergonomics but
must remain independently filterable because their platform and executable assumptions differ.

The extraction decision for diff-review is **go**.

## Acceptance Criteria

- [ ] Create `~/git/pi/pi-workbench` using the shared package convention.
- [ ] Package `diff-review`, `repo-todos`, `agent-journal`, `cwd-editor`, and `attach-screenshot` as separate manifest resources.
- [ ] Preserve `/diff`, `/diff-dirty`, `/diff-staged`, `/diff-latest`, `/diff-vs-master`, custom revspecs, and submodule behavior.
- [ ] Preserve existing overlays, shortcuts, editor export, and screenshot queue behavior.
- [ ] Imports/dependencies use `@earendil-works/*` and current Pi conventions.
- [ ] README documents Linux/macOS support and optional requirements such as Git, `$EDITOR`/`$VISUAL`, `sxiv`, and clipboard tools.
- [ ] Missing optional executables fail within the affected command rather than breaking Pi startup or unrelated workbench resources.

## Affected Files

- `pi/extensions/diff-review.ts` — source to migrate.
- `pi/extensions/repo-todos.ts` — source to migrate.
- `pi/extensions/agent-journal.ts` — source to migrate.
- `pi/extensions/cwd-editor.ts` — source to migrate.
- `pi/extensions/attach-screenshot.ts` — source to migrate.
- `pi/extensions/shared/editor-text.ts` — shared helper to relocate appropriately.
- New `~/git/pi/pi-workbench` repository — canonical implementation.

## E2E Spec

GIVEN Pi with selected workbench resources enabled on Linux or macOS
WHEN I review a Git diff, browse project todos/journal entries, open the editor, or attach screenshots
THEN enabled workflows retain current behavior and unavailable optional tools do not affect unrelated resources.

## Notes

This package may be installed globally on personal machines or selectively per project; it is not a
required universal default.
