## Context

After communication primitives and optional tmux spawning exist, we can automate
an iterative reviewer/worker loop that currently requires manual relay by the
human.

This task focuses on practical orchestration using channel messages and review
artifacts, not on building a generalized workflow engine.

## Acceptance Criteria

- [ ] There is a loop entrypoint (command/tool) that orchestrates reviewer and
      worker rounds over the communication channel.
- [ ] Reviewer results are represented as structured messages with review file
      path and severity summary.
- [ ] Worker receives actionable findings and reports fix status back through
      the channel.
- [ ] Loop termination conditions are explicit (pass, max rounds, failure,
      cancellation).
- [ ] Human can inspect loop progress and intervene from the main frame when
      needed.
- [ ] E2E coverage demonstrates at least one full reviewer→worker→reviewer
      cycle with artifact handoff.

## Affected Files

- `pi/extensions/agent-comm/` — loop orchestration module
- `pi/README.md` — reviewer-worker automation docs
- `test/e2e/pi/` — loop workflow checks

## E2E Spec

GIVEN a change set and two communication-capable agents (reviewer and worker)
WHEN the reviewer-worker loop is started
THEN rounds proceed via channel messages and review artifacts until pass,
configured stop condition, or explicit human cancellation

## Notes

Prefer explicit message contracts over freeform narrative parsing. Keep defaults
safe and observable.
