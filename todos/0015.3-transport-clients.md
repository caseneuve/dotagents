## Context

Client-side transports that implement `MessageTransport` and connect to the
relay server. UDS for local (same machine), HTTP/SSE for remote (cross-machine).
Both are push-based — no polling needed.

## Acceptance Criteria
- [ ] `UdsTransport` connects to local UDS socket
- [ ] `UdsTransport.subscribe(channel, callback)` — receives push messages
- [ ] `UdsTransport.publish/read/ack` — send via UDS
- [ ] `HttpTransport` connects to relay HTTP endpoint
- [ ] `HttpTransport.subscribe(channel, callback)` — SSE stream, auto-reconnect
- [ ] `HttpTransport.publish/read/ack` — REST calls via fetch()
- [ ] Both transports handle connection loss and reconnection gracefully
- [ ] Tests against relay server (integration)

## Affected Files
- `pi/extensions/agent-channel/transports.ts` — add UdsTransport, HttpTransport
- `pi/extensions/agent-channel/transports.test.ts` — integration tests

## Notes
- `MessageTransport` may need a `subscribe(channel, callback)` method for push
- The subscribe method replaces the poller's role for push-based transports
- `EventSource` API handles SSE reconnection with `Last-Event-ID` automatically
- UDS reconnection: simple retry with backoff
