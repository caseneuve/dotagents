## Context

Compact inline rendering is not enough for debugging or trust. We also need a
way for humans to inspect past workflow runs, see round-by-round handoffs, and
optionally drill down into raw transcript details without injecting all of that
noise into the parent LLM context.

This task should add the inspection surface for completed or in-progress runs,
including how traces are persisted in tool details and when full raw traces
should spill to files.

## Acceptance Criteria

- [ ] Completed subagent runs can be inspected after the fact through a
      user-facing command or viewer, not only while the tool is currently
      streaming.
- [ ] The inspection surface exposes at least the workflow overview, per-round
      summaries, handoff payloads, and parsed review/work results.
- [ ] The framework persists enough trace data in tool result details for the
      inline renderer and commands to work across saved sessions and forks.
- [ ] If raw transcripts or tool outputs are too large for session details,
      the framework stores them under a canonical project-local trace directory
      such as `.pi/subagents/traces/` and links to them from the structured
      trace.
- [ ] There is a discoverable way to inspect loaded subagent definitions and
      their source/model/tool policy so humans can audit what a workflow used.
- [ ] Failure cases make the unresolved findings or failure reason easy to
      inspect without reading raw JSON unless the user explicitly asks for it.

## Affected Files

- `pi/extensions/subagents/commands.ts` — inspection and listing commands
- `pi/extensions/subagents/viewer.ts` — detailed trace or overlay viewer
- `pi/extensions/subagents/persistence.ts` — detail vs file-backed trace
  storage rules
- `pi/extensions/subagents/registry.ts` — agent-definition inspection helpers
- `pi/README.md` — command and inspection workflow docs
- `test/unit/pi/subagents/` — persistence and trace-shaping tests
- `test/e2e/pi/` — inspection workflow tests across saved session state

## E2E Spec

GIVEN a completed subagent workflow in a saved session
WHEN the user runs the inspection command for that workflow
THEN they can review the workflow overview, per-round summaries, and handoff
results without rerunning the workflow

GIVEN a workflow whose raw transcript exceeds the compact session detail budget
WHEN the framework persists its trace
THEN the saved session retains the structured summary and points to a durable
transcript file for the full raw details

## Notes

Prefer one clear command path such as `/subagents` and `/subagent-open <id>`.
Keep the machine-readable trace format stable so future custom viewers or export
tools can reuse it. Spill files should live under a deterministic project-local
path such as `.pi/subagents/traces/`, remain durable until explicit cleanup, and
be easy to redirect to temporary project roots in containerized tests.
