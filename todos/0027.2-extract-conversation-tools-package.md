## Context

Assistant outline belongs with the other extensions that operate on Pi conversation/session state.
Create one logical package containing assistant outline, last assistant block, bookmarks, and
session notes while keeping each extension independently filterable.

## Acceptance Criteria

- [ ] Create `~/git/pi/pi-conversation-tools` using the shared package convention.
- [ ] Package `assistant-outline`, `last-assistant-block`, `bookmark`, and `session-notes` as separate manifest resources.
- [ ] Preserve outline/commands modes, section comments, editor export, shortcuts, bookmarks, and branch-aware notes.
- [ ] Avoid a hard dependency on runtime-ui; status and branch refresh integration degrades gracefully when that package is absent.
- [ ] Imports/dependencies use `@earendil-works/*` and current Pi conventions.
- [ ] README documents local/Git installation plus OS-specific optional clipboard/editor tools.
- [ ] Package is suitable as a global default while allowing individual resources to be disabled.

## Affected Files

- `pi/extensions/assistant-outline/*` — source to migrate.
- `pi/extensions/last-assistant-block.ts` — source to migrate.
- `pi/extensions/bookmark.ts` — source to migrate.
- `pi/extensions/session-notes.ts` — source to migrate.
- New `~/git/pi/pi-conversation-tools` repository — canonical implementation.

## E2E Spec

GIVEN Pi with the conversation package installed
WHEN I navigate an assistant response, reuse its last block, bookmark a prompt, and manage session notes
THEN each workflow works independently and optional runtime UI integration does not become a hard dependency.

## Notes

Keep each capability separately configurable even though they share one repository and package.
